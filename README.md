# Poker Log Viewer (WPF + C# + CMake)

## Описание проекта

Desktop-приложение для просмотра и анализа покерных логов в формате JSON.

Приложение позволяет:
- выбрать папку с логами
- рекурсивно просканировать JSON файлы
- распарсить данные о покерных раздачах
- отобразить данные в иерархическом интерфейсе:
  TableName → HandID → Details

---

## Технологии

- C#
- .NET 8
- WPF (без сторонних UI библиотек)
- MVVM архитектура
- System.Text.Json
- CMake (генерация Visual Studio solution)
- Windows 11 (x64)

---

## Архитектура

Проект разделён на слои:

- Views — WPF интерфейс (XAML)
- ViewModels — MVVM логика (INotifyPropertyChanged, ICommand)
- Services — фоновое сканирование файлов (Thread-based)
- Models — структуры данных (PokerHand)

---

## Многопоточность

Для фоновой обработки используется:

- System.Threading.Thread

Особенности:
- сканирование файлов выполняется в отдельном потоке
- UI обновляется через Dispatcher (BeginInvoke/Invoke)
- использование async/await, Task, ThreadPool не используется

---

## Безопасность UI

- ObservableCollection обновляется только из UI потока
- все изменения данных из фонового потока проходят через Dispatcher
- исключено прямое изменение UI из фонового потока

---

## Формат входных данных

JSON файл содержит массив покерных раздач:

[
  {
    "HandID": 123456789,
    "TableName": "Berlin #01",
    "Players": ["Jack", "Henry", "Daniel"],
    "Winners": ["Jack"],
    "WinAmount": "1 000,00$"
  }
]

## Сборка проекта (CMake)

### Конфигурация проекта

cmake -S . -B build -G "Visual Studio 17 2022"

---

### Сборка проекта

cmake --build build --config Release

---

### Особенности CMake

В проекте CMake используется как:

- генератор Visual Studio solution (.sln)
- обёртка над dotnet build
- средство обеспечения воспроизводимой сборки

Фактическая сборка C# проекта выполняется через:

dotnet build PokerLogViewer.csproj -c Release

## Функциональность

Пользователь может:

- выбрать папку с логами
- запустить сканирование
- видеть статус выполнения
- просматривать данные:
  - список таблиц (TableName)
  - список раздач (HandID)
  - детали выбранной раздачи

---

## Обработка ошибок

- битые JSON файлы пропускаются без остановки приложения
- ошибки логируются в статус UI
- приложение продолжает работу при некорректных данных

---

## Принятые архитектурные решения

- MVVM для разделения UI и логики
- Thread для фоновой обработки (строго по ТЗ)
- Dispatcher для безопасного обновления UI
- Dictionary используется как in-memory индекс данных для быстрого доступа
